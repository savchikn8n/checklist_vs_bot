# Checklist Bot

Telegram Mini App с чеклистом по дням недели и 4-недельному циклу.

## Что уже реализовано

- При открытии мини-приложения показывается текущий день недели.
- Неделя определяется по циклу из 4 недель.
- Первая неделя цикла начинается с первого понедельника месяца.
- Крестики из исходной таблицы считаются задачами, которые на этой неделе делать не нужно.
- Выполненные пункты отмечаются чекбоксами и сохраняются в браузере Telegram.
- Есть webhook для Telegram-бота, который по `/start` отправляет кнопку открытия мини-приложения.

## Структура проекта

- `app/page.tsx`:
  Главная страница мини-приложения.
- `components/checklist-app.tsx`:
  Весь интерфейс чеклиста, логика чекбоксов и отображение задач.
- `lib/checklist-data.ts`:
  Все пункты чеклиста по дням недели. Здесь редактируются тексты задач и недели, на которых задача пропускается.
- `lib/week-utils.ts`:
  Логика вычисления первой недели месяца и текущей недели цикла.
- `app/api/telegram/webhook/route.ts`:
  Webhook для Telegram. Он отвечает на `/start` и отправляет кнопку запуска мини-приложения.
- `.env.example`:
  Пример переменных окружения.

## Как поменять задачи

Открыть `lib/checklist-data.ts`.

У каждой задачи есть:

- `text`: текст задачи.
- `offWeeks`: список недель, когда задачу делать не нужно.

Пример:

```ts
{
  id: "monday-storage",
  text: "Убираемся в кладовке",
  offWeeks: [1, 3, 4]
}
```

Это значит:

- на 1-й, 3-й и 4-й неделе задача скрыта из активного списка;
- на 2-й неделе задача будет активной.

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env.local` на основе `.env.example`.

3. Запустить проект:

```bash
npm run dev
```

4. Открыть:

```text
http://localhost:3000
```

## Что вставить в `.env.local`

```env
BOT_TOKEN=токен_бота_из_BotFather
APP_URL=https://ваш-проект.vercel.app
```

Где взять значения:

- `BOT_TOKEN`:
  Открыть `@BotFather` в Telegram, создать бота через `/newbot`, скопировать токен.
- `APP_URL`:
  Это адрес проекта после деплоя на Vercel.

## Как связать с Telegram

### 1. Создать бота

В `@BotFather`:

- выполнить `/newbot`;
- задать имя;
- задать username, который заканчивается на `bot`;
- сохранить токен.

### 2. Задать домен мини-приложения

В `@BotFather`:

- открыть `/mybots`;
- выбрать бота;
- `Bot Settings`;
- `Menu Button`;
- указать URL вашего Vercel-проекта.

### 3. Подключить webhook

После деплоя выполнить:

```bash
curl "https://api.telegram.org/bot<ВАШ_BOT_TOKEN>/setWebhook?url=https://<ВАШ_ДОМЕН>/api/telegram/webhook"
```

После этого бот на команду `/start` будет присылать кнопку `Открыть чеклист`.

```bash
git remote add origin https://github.com/USERNAME/REPO.git

```

## Как выложить на Vercel

1. Зайти в Vercel.
2. Нажать `Add New Project`.
3. Выбрать GitHub-репозиторий.
4. В настройках проекта добавить переменные окружения:
   - `BOT_TOKEN`
   - `APP_URL`
5. Нажать `Deploy`.

## Ограничение Telegram

Полностью автоматически открыть мини-приложение сразу при входе в чат бот не может.

Реально доступный вариант в Telegram такой:

- пользователь открывает бота;
- бот показывает кнопку `Открыть чеклист`;
- по нажатию открывается Mini App.
